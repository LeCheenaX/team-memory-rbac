# 运行时适配器模块

运行时适配器模块把 Hermes、OpenClaw、HTTP、MCP、local client、CLI 和 host lifecycle 接到核心 RBAC/Memory/History/Sync。适配器不得复制权限判断或记忆领域规则；它们只做认证、schema 暴露、参数转换、日志和 transport。

代码出处：`src/adapters/runtime/gateway.ts`、`src/adapters/runtime/operations.ts`、`src/adapters/runtime/development-stack.ts`、`src/adapters/http/server.ts`、`src/adapters/http/client.ts`、`src/adapters/mcp/stdio-server.ts`、`src/adapters/hermes/http_client.py`、`src/adapters/hermes/contract_schema.py`、`src/adapters/hermes/session_context.py`、`src/adapters/hermes/__init__.py`、`adapters/hermes/README.md`、`adapters/hermes/team_memory_plugin/__init__.py`、`adapters/hermes/team_memory_plugin/plugin.yaml`、`src/adapters/openclaw/plugin.ts`、`adapters/openclaw/openclaw.plugin.json`、`src/adapters/agent/transports.ts`、`src/adapters/local/client.ts`、`src/adapters/local/main-agent-session.ts`、`src/adapters/local/session-store.ts`、`src/adapters/claude-code/hooks.ts`、`src/adapters/lifecycle/host-memory.ts`、`src/adapters/postgres/cloud-memory-schema.sql`、`src/index.ts`、`src/agent/tools.ts`、`src/agent/session.ts`、`src/agent/index.ts`。

本模块术语：

| 名称 | 含义 |
| --- | --- |
| adapter | 把某个 host/protocol 接到核心服务的薄层。它不拥有领域规则。 |
| transport | HTTP、stdio、local function call 等传输方式。 |
| Gateway | 所有普通工具/API 请求进入核心模块前的可信入口。 |
| provider/plugin | Host 侧暴露给 Agent 的工具集合，例如 Hermes 的 `team_memory_*`。 |
| host lifecycle | host 在用户回合前后自动执行的 recall/capture/evidence 记录流程。 |
| trusted context | 由 token/session/host 解析出的用户、Agent、root、scope 信息，不来自普通 payload。 |

## TeamMemoryGateway

Gateway 是 tool/API 的可信入口。

职责：

1. 认证 token/session。
2. 拒绝 payload 中的 identity override。
3. 调用[[RBAC模块.md#PermissionRouter]]。
4. 校验 Agent-facing schema。
5. 将 capture/search/catalog/resource 操作转发到核心服务。

输入/输出字段含义：

| 字段 | 含义 |
| --- | --- |
| `operations` | Agent-facing capture 操作列表，结构见[[记忆模块.md#MemoryCaptureOperation]]。 |
| `target` / `patch` | 单对象更新形式；普通语义写入优先使用 `operations[]`。 |
| token/session | transport 层提供的认证材料，不在普通 JSON payload 里。 |
| `commitIds` | 写入成功后 History 生成的 commit id 列表。 |
| `extra` | 非固定、可扩展的诊断/结果元数据。 |

示例调用：

```json
{
  "operations": [
    {
      "target": "memory_entity",
      "op": "create",
      "properties": {
        "name": "Riverfront",
        "desc": "Nova CRM customer churn warning pilot."
      }
    }
  ]
}
```

预期结果：Gateway 不要求 Agent 传 `clientMutationId`；低层写入时由 Gateway/History 生成幂等键。

## HermesTeamMemoryProvider

Hermes provider 暴露 `team_memory_search`、`team_memory_catalog`、`team_memory_capture`、`team_memory_lifecycle_log`。提示词和交互流程必须写在同一份测试文档中，不能分离；本地场景见[[本地手动测试.md#Scenario 正文]]，多端场景见[[多端同步手动测试.md#多端同步手动测试]]。

工具含义：

| 工具 | 含义 |
| --- | --- |
| `team_memory_search` | 调用核心 `memory.search`，只读检索。 |
| `team_memory_catalog` | 调用核心 catalog/list，返回 L3 目录。 |
| `team_memory_capture` | 调用核心 Agent-facing capture，写入 durable semantic memory。 |
| `team_memory_lifecycle_log` | 读取 host/plugin 调试日志，不写记忆。 |

示例 Tool：

```text
team_memory_capture
```

Input JSON：

```json
{
  "operations": [
    {
      "target": "memory_entity_branch",
      "op": "create",
      "subject": "Riverfront",
      "properties": {
        "name": "Riverfront receives OpenClaw ticket summaries",
        "desc": "OpenClaw pushes customer support ticket summaries to Riverfront."
      }
    }
  ]
}
```

预期结果：

```json
{
  "status": "captured",
  "commitIds": ["commit:generated"]
}
```

## Scenario4InitialRecallPath

用于 `Scenario 4: Initial Recall Before Answering` 的真实路径。

路径术语：

| 名称 | 含义 |
| --- | --- |
| initial recall | Agent 在回答前先检索长期记忆。 |
| lifecycle recall | host 自动注入上下文时调用的 recall；不同于 Agent 主动 search。 |
| relation graph | `MemoryRelation` 组成的对象关系图，用于检索扩展。 |
| reranking | BM25/vector/relation 候选合并后的排序阶段。 |

发送给 Hermes：

```text
/new
我昨天跟你说的那个“流失预警试点”到底叫什么正式项目名？它为什么会跟 OpenClaw 扯上关系？我现在要写周报，先帮我用两句话回忆一下。
```

Agent：

1. Receive user prompt.
2. Call `team_memory_search` with natural-language query and optional limit.
3. Answer from returned branches, resources, chunks, and relations.

Tool：

```text
team_memory_search
```

Input JSON schema：见[[记忆模块.md#MemorySearchInput]]。

Input JSON：

```json
{
  "query": "流失预警试点 正式项目名 OpenClaw 客服工单摘要",
  "limit": 10,
  "layer": "L2",
  "tagsAny": ["project:riverfront", "openclaw"]
}
```

Real path：

1. `team_memory_search` -> `HermesTeamMemoryProvider.search`.
2. Provider calls `memory.search`; lifecycle recall may call `/host/hermes/recall` only for host lifecycle context.
3. Gateway authenticates token and rejects identity override fields.
4. `PermissionRouter` authorizes `search:memory_entity`.
5. `MemoryRetrievalAdapter` searches BM25, vectors, SQL active view, and relation graph.
6. `MemoryRetrievalAdapter` 按最终融合分数降序执行 top-P selection，返回条数固定为 `min(N_top-P, N_limit)`。
7. lifecycle recall 的 `formatInjectedMemoryContext` 完整接收这批结果，以所有返回条目的总分 `totalScore = sum(item.score)` 为分母，按 `item.score / totalScore` 稳定降序生成模型上下文；同分保持原相对顺序，且格式化层不得再设置独立条数上限或二次截断。

Memory result：

```json
{
  "rootEntityId": "session-root",
  "branchRef": "main",
  "items": [
    {
      "kind": "entity",
      "entity": { "name": "Riverfront" },
      "branch": {
        "name": "Riverfront receives OpenClaw ticket summaries",
        "desc": "Riverfront is related to OpenClaw because OpenClaw pushes customer support ticket summaries to Riverfront.",
        "tags": ["project:riverfront", "openclaw"],
        "extra": {}
      },
      "relations": [
        {
          "type": "relates_to",
          "subject": "Riverfront",
          "object": "OpenClaw"
        }
      ],
      "evidence": [],
      "score": 1,
      "origin": "cloud_active"
    }
  ]
}
```

验收结果：

- 回答必须说正式项目名是 Riverfront。
- 回答必须说它和 OpenClaw 的关系是 OpenClaw 推送客服工单摘要。
- 本 scenario 不允许调用 `team_memory_capture`。
- 验收结果必须放在返回结果之后。

## HttpServerClient

HTTP server/client 是跨进程 adapter。

字段含义：

| 字段 | 含义 |
| --- | --- |
| `method` | HTTP 方法。 |
| `path` | HTTP API 路径。 |
| `body` | 传给 Gateway 的 JSON 参数。 |
| auth header/session | HTTP 层认证材料，不允许由 body 覆盖身份。 |

示例请求：

```json
{
  "method": "POST",
  "path": "/memory/search",
  "body": {
    "query": "Riverfront OpenClaw",
    "limit": 10
  }
}
```

预期结果：HTTP 层做 auth 和 JSON schema validation，再进入 Gateway。

## McpStdioServer

MCP server 暴露同一组工具 schema。

字段含义：

| 字段 | 含义 |
| --- | --- |
| tool name | MCP 暴露的工具名，例如 `team_memory_search`。 |
| input schema | 工具参数 JSON schema，必须与 Gateway 的 Agent-facing schema 对齐。 |
| stdio transport | MCP 进程通信通道，不改变权限和记忆语义。 |

示例工具：`team_memory_search`。

预期结果：MCP tool input 与[[记忆模块.md#MemorySearchInput]]一致。

## OpenClawPlugin

OpenClaw adapter 只能通过 Team Memory 工具写入授权事实或导入资源，不得绕过 Gateway。

字段含义：

| 字段 | 含义 |
| --- | --- |
| OpenClaw 摘要 | 外部系统产出的客服工单摘要，作为 L1 resource 或 L2 fact 来源。 |
| evidence link | 摘要证据和语义事实之间的 `refers_to` relation。 |
| Team Memory 工具 | OpenClaw 唯一允许调用的记忆入口。 |

示例调用：OpenClaw 推送客服工单摘要后，写入 branch 和 evidence link。

预期结果：写入前经过 RBAC；原始摘要走[[资源摄取模块.md#Resource]]，语义事实走[[记忆模块.md#MemoryEntityBranch]]。

## LocalAgentSession

Local session store 保存 host/session/token 的 trusted context。

字段含义：

| 字段 | 含义 |
| --- | --- |
| `host` | 会话来自哪个 host。 |
| `sessionId` | 本地会话 id，用于审计和日志关联。 |
| `rootEntityId` | 本 session 绑定的记忆 root。 |
| token | 本地保存/读取的认证凭据；不由 Agent payload 传入。 |

示例结果：

```json
{
  "host": "hermes",
  "sessionId": "20260714_062637_15f6a2",
  "rootEntityId": "root:riverfront"
}
```

预期结果：该 context 参与 RBAC 和 History provenance；Agent payload 不再重复提供。

## HostLifecycleAdapter

Host lifecycle adapter 捕获对话成功/失败路径，作为 L1 lifecycle evidence。它可以触发 extraction，但 extraction 产出的语义写入仍必须符合[[记忆模块.md#AgentFacingCaptureInput]]。

字段含义：

| 字段 | 含义 |
| --- | --- |
| lifecycle evidence | host 自动记录的 L1 证据，例如一次成功/失败路径摘要。 |
| extraction | 从 L1 证据中抽取 L2/L3 语义记忆的过程。 |
| `kind` | evidence 或对象类别，例如 `resource_chunk`。 |
| `description` | lifecycle evidence 的说明，不得被当成 `MemoryEntity.name`。 |
| `parentAgentId` | 只有 host hook/provider 明确提供父子 Agent 关系时才记录；普通 lifecycle payload 不要求 Agent 自报。 |
| `delegationId` | delegation/task 的稳定授权句柄，可在没有具体 sub-agent id 时单独存在。 |
| `extra` | lifecycle result 中可变 metadata 的唯一扩展位置。 |

示例结果：

```json
{
  "kind": "resource_chunk",
  "name": "chunk:host-capture-revision:0",
  "description": "lifecycle_l1_evidence"
}
```

预期结果：L1 证据可以被 `refers_to` 关系链接；不能把整段原始提示词当 `MemoryEntity.name`。

稳定 lifecycle capture result 字段：`status`、`outcome`、`resourceId`、
`revisionId`、`chunkIds`、`extractionCandidates`、`commitIds`、`extra`。
Hermes、OpenClaw、Claude Code、Codex 等 host 差异必须放在 trusted
session/context 或 result `extra` 中，不放进普通 Agent-facing capture payload。
