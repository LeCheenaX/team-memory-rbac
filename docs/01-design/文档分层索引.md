# 文档分层索引

本索引定义除根级 PRD 和总设计说明外，其余 canonical 文档的三层归属。新增和修改规范时必须放入下列三层之一：设计文档、配置文档、测试文档。

## 第一层：具体设计文档

具体设计文档负责覆盖代码、数据结构、接口、调用示例和预期结果。

Canonical module docs：

- [[RBAC模块.md#RBAC 模块]]
- [[记忆模块.md#记忆模块]]
- [[操作记录模块.md#操作记录模块]]
- [[同步模块.md#同步模块]]
- [[资源摄取模块.md#资源摄取模块]]
- [[运行时适配器模块.md#运行时适配器模块]]
- [[代码覆盖索引.md#代码覆盖索引]]

ADR 归属：

- `docs/adr/0001-typescript-core-python-hermes-adapter.md`：归入[[运行时适配器模块.md#HermesTeamMemoryProvider]]。
- `docs/adr/0002-cloud-authority-local-pending-overlay.md`：归入[[同步模块.md#WorkingReplica]] 和[[操作记录模块.md#MemoryCommit]]。
- `docs/adr/0003-runtime-agent-memory-integration-modes.md`：归入[[运行时适配器模块.md#TeamMemoryGateway]]。
- `docs/adr/0004-host-memory-lifecycle-adapters.md`：归入[[运行时适配器模块.md#HostLifecycleAdapter]]。

## 第二层：配置文档

配置文档负责运行模式、配置字段、启动入口和依赖检查。

Canonical config docs：

- [[配置文档-local.md#配置文档-local]]
- [[配置文档-多端同步.md#配置文档-多端同步]]

配置文档必须从操作者视角写 PowerShell/CLI 步骤、交互提示、通过条件和日志位置。不得用“手工编辑 JSON 字段”替代 setup 流程。

## 第三层：手动测试文档

手动测试文档负责人工执行流程、scenario、预期 tool call、memory result 和验收。

Canonical manual test docs：

- [[本地手动测试.md#本地手动测试]]
- [[多端同步手动测试.md#多端同步手动测试]]

测试文档必须自包含完整 scenario、发送给 Hermes 的文本、预期工具调用、日志对账和通过条件。不得引用 legacy 文档作为测试依据。

## 根级例外

以下文档是根级计划/总设计，不属于三层之一，但必须引用三层文档作为细节来源：

- `DESIGN-NOTES.md`
- `.scratch/core-development-plan/PRD.md`
- `.scratch/core-development-plan/issues/*.md`

验收规则：

- 任何 Agent-facing capture 示例必须引用[[记忆模块.md#AgentFacingCaptureInput]]。
- 任何模块代码文件必须能在[[代码覆盖索引.md#代码覆盖索引]]找到出处。
- 任何 Hermes scenario 必须把 prompt、Agent workflow、Tool、Input JSON、Real path、Memory result、验收结果写在同一 scenario 中。
- `docs/legacy/` 只保存历史材料，不是 canonical 文档，不参与验收入口。
