# 多端同步手动测试

本文件是 Test 2 的唯一维护入口：一个 Team Memory server authority，至少两个真实 Hermes client 容器连接同一 server。每个 Scenario 必须在本节内同时给出发送内容或命令、Expected input JSON、Expected result JSON 和验收点。

配置步骤见[[配置文档-多端同步.md#配置文档-多端同步]]。

## 验收材料

- Hermes A transcript。
- Hermes B transcript。
- Hermes read-only transcript。
- Hermes A/B/read-only tool-call logs。
- Team Memory service logs。
- Server admin CLI 输出。
- conflict list/resolve 输出。
- 最终 recall 的 memory result。

PowerShell、curl、npm 输出只能证明 setup/admin 操作，不能替代 Hermes 对话验收。

## 执行前检查

分别向 Hermes A 和 Hermes B 发送：

```text
Show the active memory provider, my Team Memory identity, and the memory tools visible to this session.
```

Expected input JSON:

```json
{
  "tool": "team_memory_catalog",
  "input": {}
}
```

Expected result JSON:

```json
{
  "provider": "team_memory",
  "rootEntityId": "root:test2-server",
  "endpoint": "http://service:3000",
  "sessionIssuedByServer": true
}
```

验收点：A/B 都指向同一 server authority；只看到 server-issued session 允许的工具。

## Scenario 1: Server Authority Health

PowerShell：

```powershell
curl.exe http://127.0.0.1:3000/live
curl.exe http://127.0.0.1:3000/ready
curl.exe -H "Authorization: Bearer $env:ADMIN_TOKEN" http://127.0.0.1:3000/identity
```

Expected input JSON:

```json
{
  "requests": [
    { "method": "GET", "path": "/live" },
    { "method": "GET", "path": "/ready" },
    { "method": "GET", "path": "/identity", "auth": "ADMIN_TOKEN" }
  ]
}
```

Expected result JSON:

```json
{
  "live": { "status": "ok" },
  "ready": { "status": "ready" },
  "identity": {
    "userId": "user:test2-admin",
    "rootEntityId": "root:test2-server",
    "admin": true
  }
}
```

验收点：service 是唯一 authority；Hermes client 不直接写本地 authority。

## Scenario 2: Permission Administration Boundary

向 Hermes A 发送：

```text
Try to configure a new Team Memory user or agent permission using my current client credential. This should fail unless this session is a server-authenticated human admin. Report the exact denial.
```

Expected input JSON:

```json
{
  "actor": "hermes-a",
  "attemptedAdminAction": "configure_user_or_agent_permission",
  "credentialKind": "ordinary_hermes_agent"
}
```

Expected result JSON:

```json
{
  "status": "denied",
  "reason": "admin_permission_required",
  "adminMutationApplied": false
}
```

然后用 server admin token 执行：

```powershell
$env:TEAM_MEMORY_TOKEN = $env:ADMIN_TOKEN
npm.cmd run team -- --config config/team-memory.server-local.json members list
npm.cmd run team -- --config config/team-memory.server-local.json delegations list
```

Expected admin input JSON:

```json
{
  "commands": [
    { "tool": "team_cli", "action": "members list", "auth": "ADMIN_TOKEN" },
    { "tool": "team_cli", "action": "delegations list", "auth": "ADMIN_TOKEN" }
  ]
}
```

Expected admin result JSON:

```json
{
  "status": "success",
  "membersListed": true,
  "delegationsListed": true
}
```

验收点：普通 Hermes agent/client token 不能配置权限；server admin token 可以。

## Scenario 3: Shared Server Memory Capture

向 Hermes A 发送：

```text
Remember this as a server-backed shared memory: "Hermes A says Test 2 uses one Team Memory server, server-side permission setup, and multiple Hermes clients." After saving, report the capture result.
```

Expected input JSON:

```json
{
  "tool": "team_memory_capture",
  "input": {
    "operations": [
      {
        "target": "memory_entity",
        "op": "create",
        "properties": {
          "name": "Test 2 shared server memory",
          "desc": "Hermes A says Test 2 uses one Team Memory server, server-side permission setup, and multiple Hermes clients.",
          "tags": ["test:test-2", "server-backed", "hermes-a"]
        }
      },
      {
        "target": "memory_entity_branch",
        "op": "create",
        "subject": "Test 2 shared server memory",
        "properties": {
          "name": "Hermes A shared server fact",
          "desc": "Hermes A says Test 2 uses one Team Memory server, server-side permission setup, and multiple Hermes clients.",
          "tags": ["test:test-2", "server-backed", "hermes-a"]
        }
      }
    ]
  }
}
```

Expected result JSON:

```json
{
  "status": "captured",
  "commitIds": ["commit:generated"],
  "items": [
    { "kind": "memory_entity", "name": "Test 2 shared server memory" },
    { "kind": "memory_entity_branch", "name": "Hermes A shared server fact" }
  ]
}
```

验收点：不得包含 `clientMutationId`、`entityName`、`title`、`relationType`、旧 op；`commitIds` 非空。

## Scenario 4: Cross-Client Recall

向 Hermes B 发送：

```text
/new
Recall what Hermes A saved about Test 2. Use Team Memory before answering and include the memory evidence you found.
```

Expected input JSON:

```json
{
  "tool": "team_memory_search",
  "input": {
    "query": "Hermes A Test 2 one Team Memory server server-side permission setup multiple Hermes clients",
    "limit": 10,
    "layer": "L2",
    "tagsAny": ["test:test-2", "server-backed"]
  }
}
```

Expected result JSON:

```json
{
  "items": [
    {
      "kind": "entity",
      "entity": { "name": "Test 2 shared server memory" },
      "branch": {
        "name": "Hermes A shared server fact",
        "desc": "Hermes A says Test 2 uses one Team Memory server, server-side permission setup, and multiple Hermes clients."
      },
      "origin": "cloud_active"
    }
  ]
}
```

验收点：Hermes B 必须从 server memory 召回 Hermes A 写入内容，不依赖短期上下文。

## Scenario 5: Read-Only Client Denial

用 `HERMES_READONLY_TOKEN` 启动 read-only Hermes session，发送：

```text
Show my visible Team Memory tools. Then try to save a new memory. The write should be denied. Explain the missing permission.
```

Expected input JSON:

```json
{
  "tool": "team_memory_capture",
  "input": {
    "operations": [
      {
        "target": "memory_entity",
        "op": "create",
        "properties": {
          "name": "Read-only write attempt",
          "desc": "This write should be denied."
        }
      }
    ]
  }
}
```

Expected result JSON:

```json
{
  "status": "denied",
  "reason": "missing_permission",
  "missingPermissions": ["write:memory_entity"],
  "commitIds": []
}
```

验收点：read-only client 不产生 commit。

## Scenario 6: Authorized Sync State

向 Hermes A 发送：

```text
Check the server sync state for my authorized root. Do not include data from any other root. Tell me the commit watermark or head commit id you see.
```

Expected input JSON:

```json
{
  "action": "read_authorized_sync_state",
  "rootEntityId": "trusted-session-root",
  "requestedOtherRoot": false
}
```

Expected result JSON:

```json
{
  "status": "success",
  "rootEntityId": "root:test2-server",
  "branchRef": "main",
  "headCommitId": "commit:generated",
  "commitWatermark": "watermark:generated",
  "identityOverrideAccepted": false
}
```

验收点：只返回授权 root；不泄露其他 root；不接受 identity override。

## Scenario 7: Concurrent Conflict Creation

向 Hermes A 发送：

```text
Create a memory item named "Test 2 conflict candidate" with branch content "Hermes A keeps the target version." Keep the ids and head commit you used so Hermes B can attempt a stale concurrent update.
```

Hermes A Expected input JSON:

```json
{
  "tool": "team_memory_capture",
  "input": {
    "operations": [
      {
        "target": "memory_entity",
        "op": "create",
        "properties": {
          "name": "Test 2 conflict candidate",
          "desc": "Hermes A keeps the target version.",
          "tags": ["test:test-2", "conflict"]
        }
      },
      {
        "target": "memory_entity_branch",
        "op": "create",
        "subject": "Test 2 conflict candidate",
        "properties": {
          "name": "Test 2 conflict target version",
          "desc": "Hermes A keeps the target version.",
          "tags": ["test:test-2", "conflict", "target"]
        }
      }
    ]
  }
}
```

Hermes A Expected result JSON:

```json
{
  "status": "captured",
  "commitIds": ["commit:target"],
  "headCommitId": "commit:target"
}
```

向 Hermes B 发送：

```text
Using the stale head commit from Hermes A's first write, attempt to update the same memory item with branch content "Hermes B is the incoming version." Report whether Team Memory created a conflict.
```

Hermes B Expected input JSON:

```json
{
  "baseHeadCommitId": "commit:target",
  "operations": [
    {
      "target": "memory_entity_branch",
      "op": "create",
      "subject": "Test 2 conflict candidate",
      "properties": {
        "name": "Test 2 conflict incoming version",
        "desc": "Hermes B is the incoming version.",
        "tags": ["test:test-2", "conflict", "incoming"]
      }
    }
  ]
}
```

Hermes B Expected result JSON:

```json
{
  "status": "conflict",
  "conflicts": [
    {
      "status": "open",
      "conflictKey": "memory_entity_branch:Test 2 conflict candidate"
    }
  ],
  "silentOverwrite": false
}
```

验收点：B 的 stale write 不得静默覆盖 A；该 conflict 是并发写冲突，不是语义 `contradicts` relation。

## Scenario 8: Admin Conflict Resolution

PowerShell：

```powershell
$env:TEAM_MEMORY_TOKEN = $env:ADMIN_TOKEN
npm.cmd run team -- --config config/team-memory.server-local.json conflicts list
npm.cmd run team -- --config config/team-memory.server-local.json conflicts resolve <conflict-id> take_incoming
```

Expected input JSON:

```json
{
  "commands": [
    { "tool": "team_cli", "action": "conflicts list", "auth": "ADMIN_TOKEN" },
    { "tool": "team_cli", "action": "conflicts resolve", "conflictId": "conflict:generated", "strategy": "take_incoming", "auth": "ADMIN_TOKEN" }
  ]
}
```

Expected result JSON:

```json
{
  "status": "resolved",
  "conflictId": "conflict:generated",
  "strategy": "take_incoming",
  "newCommitId": "commit:resolution",
  "auditRecordCreated": true
}
```

验收点：只有 admin token 能 resolve；resolution 生成 commit/audit。

## Scenario 10: Final Multi-Client Log Review

向 Hermes A 发送：

```text
Summarize the Team Memory calls from this multi-client test: which client wrote, which client recalled, which client was denied, what sync/head state was checked, and how the conflict was resolved. Use logs or tool results rather than memory guesses.
```

Expected input JSON:

```json
{
  "logSources": [
    "hermes-a:/root/.hermes/team-memory-tool-calls.jsonl",
    "hermes-b:/root/.hermes/team-memory-tool-calls.jsonl",
    "service:docker compose logs service",
    "admin-cli:conflicts list/resolve output"
  ]
}
```

Expected result JSON:

```json
{
  "summary": {
    "clientAWrote": true,
    "clientBRecalled": true,
    "readOnlyDenied": true,
    "syncStateChecked": true,
    "conflictCreated": true,
    "conflictResolvedByAdmin": true
  }
}
```

验收点：summary 必须和 A/B/read-only transcripts、tool-call logs、service/admin logs 一致。

## Log 对账规则

读取 A/B logs：

```powershell
docker compose -f compose.yaml -f compose.hermes.yaml run --rm hermes-a sh -lc "cat /root/.hermes/team-memory-tool-calls.jsonl"
docker compose -f compose.yaml -f compose.hermes.yaml run --rm hermes-b sh -lc "cat /root/.hermes/team-memory-tool-calls.jsonl"
docker compose logs service
```

逐项检查：

| 场景 | A log | B log | service/admin log |
| --- | --- | --- | --- |
| Shared capture | capture success, commitIds non-empty | 不要求 | write request applied |
| Cross recall | 不要求 | search success, result contains A memory | search request authorized |
| Read-only denial | 不要求 | 不要求 | denial/no commit |
| Sync state | sync/head request authorized | 不要求 | no cross-root leak |
| Conflict | A first write success | B stale write conflict | conflict record open |
| Resolution | final recall search | final recall search | admin resolve commit/audit |

## Test 2 通过条件

全部满足才通过：

- Team Memory service stack 运行。
- 至少两个真实 Hermes client 连接同一 server。
- 权限管理只通过 server admin。
- 普通 Hermes agent/client token 不能配置权限。
- Hermes A 写入后 Hermes B 能召回。
- read-only client 不能写。
- sync/head state 只暴露授权 root。
- concurrent stale write 产生 conflict。
- conflict resolution 必须 admin 执行。
- 最终 evidence 是 transcript + tool-call logs + service/admin logs，而不是单独 shell 片段。
