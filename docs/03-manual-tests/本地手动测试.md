# 本地手动测试

本文件是 Test 1 的唯一维护入口：一个真实 Hermes local 容器、无 Team Memory HTTP server、无 sync。每个 Scenario 必须在本节内同时给出发送给 Hermes 的文本、预期工具、Expected input JSON、Expected result JSON 和验收点。

配置步骤见[[配置文档-local.md#配置文档-local]]。

## 验收材料

- Hermes conversation transcript。
- `team-memory-tool-calls.jsonl`。
- `team-memory-hooks.jsonl` 或 lifecycle/debug log。
- 每个 Scenario 的 tool input/output。
- 失败场景的 exact error。

只看 PowerShell、Python、curl、npm 输出不能判定通过。它们只用于 setup 或诊断。

## 执行前检查

按[[配置文档-local.md#0. 前置条件]]到[[配置文档-local.md#8. 启动真实 Hermes 会话]]完成配置。

启动 Hermes 后先发送：

```text
Show me which long-term memory provider is active. Then use the provider to show my Team Memory identity, the memory tools visible to this session, and the available Team Memory entity/tag catalog if a catalog tool is exposed.
```

Expected input JSON:

```json
{
  "tool": "team_memory_catalog",
  "input": {}
}
```

Expected result JSON:

```json
{
  "provider": "team_memory",
  "mode": "local",
  "rootName": "test1-local",
  "toolsVisible": ["team_memory_search", "team_memory_catalog", "team_memory_capture"],
  "serviceStarted": false
}
```

验收点：active provider 是 `team_memory`；使用 `root:test1-local`；读写能力符合当前用户；没有启动 Team Memory HTTP `service`。

| --- | --- |
| `toolName` | 与 scenario Expected input JSON 一致 |
